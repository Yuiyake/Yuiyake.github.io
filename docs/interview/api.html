    # http与https

    # 概念

    • http：超文本传输协议，用于从www服务器传输超文本到本地浏览器的传输协议。
    • https：以安全为目标的http通道，实质是http+ssl。加密的详细内容需要ssl
    • https的工作原理
      • 客户端使用https url访问服务器，请求web服务器建立ssl链接。
      • 服务器受到客户端的请求后，将网站的证书（包含公钥）返回。
      • 客户端和服务器开始协商ss链接的安全等级（加密等级）
      • 客户端浏览器通过双方协商一致的安全等级，建立会话秘钥，然后通过网站的公钥来加密会话秘钥，并 传给网站。
      • 服务器通过自己的私钥解密出会话秘钥，通过会话秘钥加密与客户端之间的通信。

    # 区别

    1. https需要ca证书，费用较高。
    2. http的信息是明文传输，简单无状态；https是具有安全性的ssl加密传输，比http协议安全。
    3. http的默认端口是80， https的默认端口是443

    # https协议优缺点

    • 优点：使用https协议可认证用户和服务器，确保数据发送到正确的客户端和服务器。
    • 缺点：https握手阶段费时，会使页面加载时间延长50%，增加10-20%的耗电。缓存不如http高效， 会增加数据开销。ssl证书费用较高且需要绑定ip，不能在同一个ip上绑定多个域名，ipv4资源不支持这种消耗

    # flex弹性布局

    • 参考文档：阮一峰的博客 (opens new window)

    # 概念：

    • 容器的属性：

      • flex-direction: row | row-reverse | column | column-reverse;（决定横纵向布局）
      • flex-wrap: no-wrap | wrap | wra-reverse;（决定换行规则）
      • flex-flow: flex-direction || flex-wrap;
      • justify-content: flex-start | flex-end | center | space-between | space-around; (水平主轴上的排列样式)
      • align-items:stretch | flex-start | flex-end | center | baseline;（竖直交叉轴上的排列样式）
      • align-content: stretch | flex-start | flex-end | center | space-between | space-around; (多条主轴的对齐方式， 比较少使用到）

    • 项目的属性：

      • order：定义项目的排列顺序，顺序越小越前。默认0
      • flex-grow：定义项目的放大比例。
      • flex-shrink：定义项目的缩小比例。
      • flex-basis：定义了在分配多余的空间，项目占据的空间大小。
      • flex：默认0 1 auto
      • align-self：允许单个项目与其他项目不同的对齐方式，可覆盖
      • align-items：默认auto，表示继承父元素的align-items，比如圣杯布局

    # cookies，sessionStorage 和 localStorage

    • 这三个都是在浏览器端存储的数据。

    # cookie:

    • 每个域名存储量较小（4k左右），所有域名的存储量都有限制，有个数限制，会随请求发送到服务器

    # sessionStorage

    • 概念：引入了“浏览器窗口”的概念。它是在同源的同窗口中始终存在的数据。也就是说只要这个浏览器窗口没有关闭， 即使刷新页面或者进入同源另一个页面，数据依然存在。在关闭窗口后sessionStorage才被销毁。
    • 同时独立打开不同的窗口，即使是同一页面，sessionStorage对象也是不同的。
    • 只在session内有效
    • 存储量更大（没有限制，但分浏览器）

    # localStorage

    • 永久存储
    • 单个域名存储量较大（6mb左右）
    • 总体数量无限制

    # DHTML

    # 介绍

    • DHTML 指动态 HTML（Dynamic HTML）。通过 DHTML，Web 开发者可控制如何在浏览器窗口中显示和定位 HTML 元素。

    # 动态样式

    • 动态样式的作用：使网页坐着改变内容的外部特征而不强制用户再次下载全部内容。

    # HTML语义化

    # 什么是语义化：

    • 方便开发者阅读和写出更优雅的代码的同时让浏览器的爬虫和机器很好地解析。

    # 如何语义化：

    • 尽量少使用无语义的标签div, span
    • 语义不明显时，既可以用div也可以用p，尽量用p，因为p默认有上下间距，对兼容特殊终端有利
    • 不要使用纯样式标签，如b, font, u等，改用css
    • 需要强调的文本，包含在strong, em中，strong是加粗(不用b)，em是斜体(不用i)
    • 用表格时，标题用caption，表头thead，主题tbody，尾tfoot，表头和单元格分开，头用th，格用td
    • 表单域要用fieldset抱起来，并用legend说明表单的用途
    • 每个input标签对应的说明文本都用label属性，在label中设置for=someId来让说明文本和相应input关联。

    # Canvas与SVG

    # Canvas

    • 支持分辨率，不支持事件处理器。弱文本渲染能力，适合图像密集型的游戏
    • canvas是逐像素渲染的
    • canvas中，一旦图形被绘制完成，它就不会继续得到浏览器的关注。如果位置改变，那么整个场景都需要 重绘，包括任何或许被图形覆盖的对象

    # SVG

    • 不支持分辨率，支持事件处理器。适合带有大型渲染区域的应用程序（如地图）
    • 复杂度搞会减慢渲染速度
    • 不适合游戏用
