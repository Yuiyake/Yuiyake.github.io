    # Flex

    # flex:1

    • flex 是 flex-grow,flex-shrink,flex-basis三个属性的简写，默认值为0 1 auto。
    • flex:1 等价于 flex: 1 1 auto;
    • flex-shrink 决定的是flex元素收缩的大小
    • flex-basis 决定的是flex在x轴上的初始大小
    • flex-grow 决定的是flex元素本身的占比大小
    • flex的详细文档 - 阮一峰老师 (opens new window)
    • 阮一峰-flex实践 (opens new window)
    • 详细属性
      • flex-direction：决定主轴是横还是纵及排列方向
      • flex-wrap：决定是否换行及第一行位置
      • flex-flow：上面两个合并的简写
      • justify-content：主轴上的水平对齐方式
      • align-items：交叉轴上的垂直对齐方式
      • align-content：多跟轴线的对齐方式

    # Grid

    # grid模型

    • 和flex不同，flex是通过确定主轴来布局的（一维），grid是通过行列布局的（二维）
    • 属性
      • grid-template-rows, grid-template-columns
      • grid-row-gap, grid-columns-gap, grid-gap
      • grid-template-areas：定义区域（一或多个单元格组成）
      • grid-auto-flow：决定格子是先行后列（默认）还是先列后行放置
      • justify-items：单元格内容的水平位置
      • align-items： ~垂直位置
      • place-items：上面两个合并的简写
      • justify-content：整个内容区在容器的水平位置
      • align-content： ~垂直位置
      • place-content：上面两个合并的简写
    • grid的详细文档 - 阮一峰老师 (opens new window)

    # 盒模型和浏览器渲染模式

    # 标准盒模型

    • margin，border，padding，context。标准盒模型就是一个块的总宽度 = width + margin + padding + border

    # 怪异盒模型

    • 也叫IE盒模型，一个块的总宽度 = width(padding+border) + margin

    # 盒模型转换

    • box-sizing:content-box | border-box | inherit
    • content-box，标准盒模型
    • border-box，怪异盒模型
    • inherit， 继承父元素box-sizing

    # 浏览器的标准模式

    • 文件开头的 DOCTYPE 决定用怪异模式还是标准模式处理。<!DOCTYPE html>就是启用标准模式。

    # 浏览器的怪异模式

    • 在怪异模式下，排版会模拟 Navigator 4 与 Internet Explorer 5 的非标准行为。

    # BFC(Block Formatting Context - 块级格式化上下文)

    # 高度塌陷

    • 为什么会有高度塌陷？ 这个问题追根到底是css盒模型。在标准文档流中，竖直方向的margin不叠加，只取较大的值作为margin。 所以当两个垂直方向的元素堆叠在一起，就会发生高度塌陷（只有一个的margin生效了）。为了解决这个问题，可以用BFC。

    # 概念

    • BFC
      • BFC是一个独立的布局环境，其中的元素布局不受外界的影响。且在一个BFC中，块盒与行盒都会 垂直沿着其父元素的边框排列。
      • BFC是一个块级元素，块级元素会在垂直方向一个接一个排列
      • BFC是页面中的一个隔离的独立容器，容器里的标签不会影响到外部标签
      • 垂直方向的距离由margin决定，属于同一个BFC的两个相邻标签外边距重叠
      • 计算BFC高度时，浮动元素也参与计算。

    # 兄弟节点的margin重叠怎么办

    • 外层overflow:hidden
    • 内层：absolute定位
    • 内层元素透明边框：border:transparent

    # display:none 和 visibility:hidden

    • 区别：visibility:hidden会保留元素的空间
    • display:none会触发回流，可以理解为元素不可见且不存在（没有空间）
    • visibility:hidden会触发重绘。可以理解为元素不可见但存在，但不触发绑定事件。
    • opacity=0：元素不可见但存在，保留空间，不影响结构，且如果元素已经绑定一些事件（如click），那么点击该区域也会触发点击事件。

    # 常见的布局单位

    • 常见的CSS布局单位 常用的布局单位包括像素(px),百分比(%),em,rem,vw/vh
    • rem+媒体查询@media rem+flexible.js/vw响应式布局。
    • rem自适应是根据html根文字大小进行等比缩放的。
    • vw/vh是根据视口大小进行等比缩放的。
