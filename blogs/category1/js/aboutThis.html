    this指向（闭包，原型链，函数声明）

    yuiyake 3/20/2022

    # 结论

    • 首先我们上结论。关于this指向的问题记住这几个结论就好

      1. 谁调用，this指向谁。
      2. 匿名函数（/立即执行函数）的this指向全局。
      3. 原型中的this指向调用他的对象不指向本身。
      4. 构造函数调用this指向实例对象。
      5. 计时器调用this指向全局

    • 然后我们再来看看下面的例子，熟悉一下这几个规则。

    # 闭包里的this

    • 闭包的作用之一就是延长函数内部的变量作用域

      • 例子一

        • 对应下面的代码，b()的作用就是让c()能读取到内部i=0这个变量。
        // 函数b能访问局部变量i，那么把b()作为返回值，外界就能读取到i这个变量。
function a() {
    var i=0; 
    function b() {
        alert(++i);
    }
    return b;
}
var c = a();
c();    // 1

    # 原型与原型链的this

    • 例子一

      function c1(name) {
    if(name) this.name = name;
}
function c2(name) {
    this.name = name;
}
function c3(name) {
    this.name = name || 'John';
}
c1.prototype.name = 'Tom';
c2.prototype.name = 'Tom';
c3.prototype.name = 'Tom';
console.log(new c1().name + "," + new c2().name + "," + new c3().name)

      function father() {
    this.num = 935;

    • 例子三

      var A = {n: 4399};
var B = function () {
  this.n = 9999;    //这里等价于b.n = 9999
}
var C = function () {
  var n = 8888; //这里的n是私有变量，不是c里的
}
B.prototype = A;
C.prototype = A;
var b = new B();
var c = new C();
A.n++;
console.log(b.n); //9999
console.log(c.n); //4400

      • 首先改写了BC的原型，所以他们的原型都拥有n:4399这个属性。
      • 通过new创建了B和C实例，这种方法调用构造函数经历4步
        1. 创建新的空obj b,c
        2. 将构造函数作用域复制给新对象（this指向新对象b，c），此时b拥有了实例属性n:9999，c执行 var n=8888，对c不影响，bc都有原型上的n=4399；
        3. 返回新对象，返回b有实例属性n=9999，c实例啥也没有
        4. 改写原型属性A.n++，因为原型上的实例是共享的，所以此时bc的原型属性n=4400;
        5. 打印。b找到了实例属性n=9999，不再向上查找。c没有找到实例属性，向上查找找到原型 属性n=4400。

    # 立即执行函数

    var myobject={
    foo:"bar",
    func:function(){
        var self=this;
        console.log(this.foo);  //bar
        console.log(self.foo);  //bar
        (function(){
            console.log(this.foo);//undefined
            console.log(self.foo);//bar
        })();
    }
};
myobject.func();
